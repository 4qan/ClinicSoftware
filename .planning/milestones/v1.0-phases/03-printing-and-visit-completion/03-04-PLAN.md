## Tasks

### Task 1: Fix @media print CSS selectors (Blocker, Tests 7+8)

**Files:** `src/index.css`, `src/components/AppLayout.tsx`

**Root cause:** The `@media print` rule in index.css targets `.sidebar` and `.app-header`, but neither class exists in the DOM. The sidebar is an `<aside>` element (no `.sidebar` class). The header is a plain `<div>` with `sticky top-0` classes (no `.app-header` class).

**Changes in AppLayout.tsx:**
- Add class `app-header` to the sticky header `<div>`

**Changes in index.css:**
- Replace `nav, header, .sidebar, .app-header, .no-print` with `aside, .app-header, .no-print`
- Add `width: 100% !important` to the existing `main` print rule so content fills the page

**Acceptance criteria:**
- Browser print preview shows only slip content
- Sidebar (`<aside>`) hidden in print
- Sticky header bar hidden in print
- Main content fills full page width in print

### Task 2: Simplify PrintVisitPage (Major, Test 6)

**File:** `src/pages/PrintVisitPage.tsx`

**Root cause:** (a) CTAs are oversized (`flex-1 px-6 py-4 text-lg`). (b) DispensarySlip is completely hidden on screen, no way to preview it.

**Changes:**
1. Shrink CTAs: replace `flex-1 py-4 text-lg` with `px-4 py-2 text-sm font-semibold`
2. Add `previewMode` state (`'prescription' | 'dispensary'`, default `'prescription'`). Render two small tab buttons (pill-style, matching Settings pattern) above the preview area to toggle between slips.
3. Both slips always in DOM. On screen: show only the active `previewMode` slip. During print: `printMode` controls which gets `print-hidden`.
4. Single print button label matches active preview tab.

**Acceptance criteria:**
- Compact controls (no oversized buttons)
- User can toggle between Prescription and Dispensary preview
- Print dialog shows only the currently previewed slip
- No buttons, tabs, or breadcrumbs in print output

### Task 3: Inline print dropdown on VisitCard (Minor, Test 5)

**File:** `src/components/VisitCard.tsx`, `src/pages/PrintVisitPage.tsx`

**Root cause:** Print link navigates to `/visit/:id/print`. User wants inline print without extra navigation.

**Changes in VisitCard.tsx:**
1. Replace single Print link with a "Print" button that toggles a dropdown
2. Dropdown shows: "Print Prescription", "Print Dispensary", "Preview"
3. Print options link to `/visit/${visit.id}/print?auto=prescription` or `?auto=dispensary`
4. "Preview" links to `/visit/${visit.id}/print` (no auto param)
5. Click-outside closes dropdown

**Changes in PrintVisitPage.tsx:**
1. Read `searchParams.get('auto')` on mount
2. If present, set `previewMode` and auto-trigger `window.print()` after data loads
3. This gives one-click print from visit card; print page serves as fallback if user cancels

**Acceptance criteria:**
- VisitCard "Print" shows dropdown with Prescription/Dispensary/Preview options
- Selecting print option navigates to print page and immediately opens browser print dialog
- Dropdown closes when clicking outside
