# Plan 02: Prescription and Dispensary Printing

## Goal
Build the dedicated print page with A5 prescription slip and compact dispensary slip, both printable via browser print dialog in Chrome/Edge.

## must_haves
- [ ] `/visit/:id/print` route loads visit data, patient info, and clinic info, showing formatted previews (PRINT-01, PRINT-02)
- [ ] "Print Prescription" button prints A5 slip with clinic header, patient info, medications, clinical notes, Rx notes, and footer (PRINT-01)
- [ ] "Print Dispensary" button prints compact slip with patient ID/name/date and medications only (PRINT-02)
- [ ] `@media print` CSS hides app chrome, `@page` sets A5 size, content does not clip or overflow (PRINT-03)
- [ ] Both buttons are prominent and clearly labeled on the print page (PRINT-04 completion)

## Tasks

<task id="03-02-01" title="PrintVisitPage route and data loading">
<context>New page at /visit/:id/print that loads all data needed for both slip types: visit + medications, patient record, and clinic info from settings. This is the container page with print buttons and preview.</context>
<steps>
1. Create `src/pages/PrintVisitPage.tsx`:
   - Extract `visitId` from URL params via `useParams()`
   - Load visit + medications via `getVisit(visitId)` from `src/db/visits.ts`
   - Load patient via `getPatient(visit.patientId)` from `src/db/patients.ts`

3. Page layout (screen view):
   - Breadcrumbs: Home > Patient > Print Prescription
   - Two large buttons at top: "Print Prescription" and "Print Dispensary Slip"
   - Below buttons: preview of the prescription slip (always visible on screen for the doctor to review before printing)
   - Buttons should be large and clearly labeled (user is not tech-savvy)
4. Button handlers:
   - Set `printMode` to the appropriate value
   - Use `setTimeout(() => window.print(), 100)` to allow DOM to update before print dialog opens
   - Listen for `afterprint` event (via `useEffect`) to reset `printMode` to null
5. Open `src/App.tsx`, add route: `<Route path="/visit/:id/print" element={<PrintVisitPage />} />`
6. Add the route inside the authenticated/layout wrapper, consistent with existing routes
</steps>

<task id="03-02-04" title="Print CSS and @media print rules">
<context>PRINT-03: Print layouts must work in Chrome/Edge print dialog. This task adds all @media print CSS rules to handle A5 page size, hide app chrome, and show only the target slip based on printMode.</context>
<steps>
1. Open `src/index.css`
2. Add `@media print` block:
   ```css
   @media print {
     /* Hide all app chrome */
     nav, header, .sidebar, .app-header, .no-print { display: none !important; }

     /* Reset body/html for print */
     body { margin: 0; padding: 0; background: white; }

     /* Page setup for A5 */
     @page { size: A5 portrait; margin: 8mm; }

     /* Hide screen-only elements */
     .screen-only { display: none !important; }

     /* Show print-only elements */
     .print-only { display: block !important; }

     /* Prevent breaking inside medication rows */
     tr { break-inside: avoid; }

     /* Ensure backgrounds print */
     * { print-color-adjust: exact; -webkit-print-color-adjust: exact; }
   }
   ```
3. In `PrintVisitPage.tsx`, implement the print visibility logic:
   - Both slips are always in the DOM
   - On screen: show PrescriptionSlip as preview, hide DispensarySlip (use `screen-only` / `hidden` classes)
   - During print (`printMode === 'prescription'`): show only PrescriptionSlip, hide everything else
   - During print (`printMode === 'dispensary'`): show only DispensarySlip, hide everything else
   - Use CSS classes toggled by `printMode` state:
     - When `printMode === 'prescription'`: DispensarySlip gets `hidden` in print, PrescriptionSlip visible
     - When `printMode === 'dispensary'`: PrescriptionSlip gets `hidden` in print, DispensarySlip visible
   - The buttons, breadcrumbs, and page wrapper get `no-print` class
4. Add `no-print` class to the AppLayout sidebar and header components (if not already hidden by the nav/header selectors)
5. Verify no `position: fixed` or `overflow: hidden` on print-visible ancestors
6. Test checklist (manual, documented in verification):
   - Chrome: Print Prescription shows A5 preview with full slip
   - Chrome: Print Dispensary shows compact medication list
   - No blank second page
   - No clipped content
   - Medication table rows don't split across pages
</steps>
