# Plan 02-02: Visit/Encounter Workflow: New Visit Page with Prescription Writing

## Objective
Deliver the New Visit page where a doctor can select a patient, write clinical notes, add medications to a prescription, and save the complete visit. This is the core clinical workflow page. Visits are fully editable and deletable (RX-05 deferred).

## must_haves
- Doctor can create a visit with clinical notes for a selected patient
- Visits are automatically timestamped (createdAt/updatedAt)
- Doctor can add medications with dosage, frequency, and duration to a visit
- Medications are saved as snapshots (not live references to drug records)
- Visit can be saved, edited, and deleted
- New Visit accessible from sidebar navigation

<task id="02-02-01" title="Visit CRUD operations">
<description>
Create `src/db/visits.ts` with functions for visit management:

- `createVisit(data: { patientId: string, clinicalNotes: string, rxNotes: string, medications: Omit<VisitMedication, 'id' | 'visitId'>[] }): Promise<string>` - Creates a visit record and all associated visitMedications in a Dexie transaction. Auto-generates UUIDs, timestamps. Returns visit ID.

- `updateVisit(visitId: string, data: { clinicalNotes: string, rxNotes: string, medications: Omit<VisitMedication, 'id' | 'visitId'>[] }): Promise<void>` - Updates visit record and replaces all medications (delete existing, insert new) in a transaction. Updates `updatedAt`.

- `deleteVisit(visitId: string): Promise<void>` - Deletes visit and all associated visitMedications in a transaction.

- `getVisit(visitId: string): Promise<{ visit: Visit, medications: VisitMedication[] } | null>` - Loads a visit with its medications, sorted by sortOrder.

- `getPatientVisits(patientId: string): Promise<Array<{ visit: Visit, medications: VisitMedication[] }>>` - Returns all visits for a patient in reverse chronological order (newest first), each with its medications.

- `removeMedicationFromVisit(medicationId: string): Promise<void>` - Deletes a single medication from a visit.

All functions use `withTimestamps()` or manual ISO timestamp generation consistent with existing patterns.
</description>

<task id="02-02-05" title="NewVisitPage implementation">
<description>
Create `src/pages/NewVisitPage.tsx`: the main visit creation page.

**Two entry modes** (determined by URL):
1. `/visit/new?patientId=xxx` - patient pre-selected (from patient profile button)
2. `/visit/new` - standalone, needs patient search

**Page layout (top to bottom):**

1. **Breadcrumbs:** Home > New Visit (or Home > Patient Name > New Visit)

2. **Patient Section** (CollapsibleSection, default open if no patient, collapsed if pre-selected):
   - If patient pre-selected: load patient by ID, show summary (name, patient ID, age, gender, contact). Collapse after load.
   - If standalone: show inline search field using usePatientSearch hook. On patient selection, show summary and collapse. Include "Register New" link to /register.

3. **Visit History** (CollapsibleSection, default collapsed):
   - Only visible after patient is selected
   - Load last 5 visits via getPatientVisits()
   - Each visit shows: date, first line of clinical notes (truncated to ~100 chars), medication count
   - "Show all" link navigates to patient profile

4. **Clinical Notes** (always visible after patient selected):
   - Single textarea with placeholder: "Complaint:\nExamination:\nDiagnosis:"
   - Auto-expanding or min-height 120px

5. **Prescription** (always visible after patient selected):
   - MedicationEntry component for adding drugs
   - MedicationList component showing added medications
   - Rx Notes textarea below: placeholder "Additional prescription notes..."

6. **Action bar** (at page bottom):
   - "Save Visit" primary button (blue-600, large, min-height 44px)
   - "Cancel" secondary button (navigates back)
   - Save is disabled until: patient selected + clinicalNotes non-empty
   - On save: call createVisit(), navigate to patient profile page

**State management:** useState for clinicalNotes, rxNotes, medications array, selectedPatient. Form-level validation before save.

**Route:** Add `/visit/new` to App.tsx router.
