# Plan 02-01: Drug Database: Schema, Seed Data, Settings UI, Autocomplete

## Objective
Deliver a fully functional drug database layer: Dexie schema with `drugs` table, ~170 pre-seeded Pakistani market medications, a settings UI for managing custom drugs, and a reusable autocomplete hook that performs under 300ms.

## must_haves
- Pre-seeded drug database loads on first run with ~170 common Pakistani medications
- Custom drugs can be added, edited, disabled, and deleted from Settings
- Drug autocomplete searches both brand and salt names with sub-300ms performance
- Custom drugs appear alongside pre-seeded drugs in autocomplete results
- Reusable ComboBox component for dropdown-with-free-text pattern

<task id="02-01-01" title="Dexie schema v2 and Drug type definitions">
<description>
Extend `ClinicDatabase` in `src/db/index.ts` with a `version(2)` migration adding three new tables: `drugs`, `visits`, `visitMedications`. Define TypeScript interfaces for `Drug`, `Visit`, and `VisitMedication` in `src/db/index.ts` (or co-located types file).

Drug interface fields: id (string, crypto.randomUUID), brandName, brandNameLower, saltName, saltNameLower, form, strength, isCustom (boolean), isActive (boolean, default true), createdAt, updatedAt.

Visit interface fields: id (string), patientId (string), clinicalNotes (string), rxNotes (string), createdAt, updatedAt.

VisitMedication interface fields: id (string), visitId (string), drugId (optional string), brandName, saltName, form, strength, dosage, frequency, duration, sortOrder (number).

Dexie indexes:
- drugs: 'id, brandNameLower, saltNameLower, isCustom'
- visits: 'id, patientId, createdAt'
- visitMedications: 'id, visitId'

Keep version(1) stores unchanged. version(2) must include ALL existing table definitions plus the new ones (Dexie requires full schema in each version).
</description>

<task id="02-01-03" title="Drug CRUD operations">
<description>
Create `src/db/drugs.ts` with functions for drug management:

- `searchDrugs(query: string): Promise<Drug[]>` - Dual index prefix search on brandNameLower and saltNameLower, filter isActive === true, deduplicate, limit 10 results. Follow the pattern from research (Dexie .where().startsWith() on both indexes).

- `addCustomDrug(data: Omit<Drug, 'id' | 'brandNameLower' | 'saltNameLower' | 'isCustom' | 'isActive' | 'createdAt' | 'updatedAt'>): Promise<string>` - Creates a new custom drug with isCustom: true, isActive: true, auto-generates lowercase fields and timestamps.

- `updateCustomDrug(id: string, data: Partial<Drug>): Promise<void>` - Updates an existing custom drug. Must verify isCustom === true before allowing edit. Auto-updates lowercase fields if brand/salt name changed.

- `toggleDrugActive(id: string): Promise<void>` - Toggles isActive flag (soft disable/enable for custom drugs).

- `deleteCustomDrug(id: string): Promise<void>` - Hard deletes a custom drug. Must verify isCustom === true.

- `getCustomDrugs(): Promise<Drug[]>` - Returns all custom drugs for the settings list.

- `getAllDrugs(): Promise<Drug[]>` - Returns all active drugs (for any future needs).
</description>

<task id="02-01-06" title="ComboBox reusable component">
<description>
Create `src/components/ComboBox.tsx`: a reusable dropdown-with-free-text input component.

Props:
- `options: string[]` - predefined options to show in dropdown
- `value: string` - controlled value
- `onChange: (value: string) => void` - change handler
- `placeholder?: string`
- `label?: string`
- `error?: string`
- `disabled?: boolean`

Behavior:
- On focus, show dropdown with all options (or filtered if text is entered)
- As user types, filter options that include the typed text (case-insensitive)
- User can select from dropdown OR type a custom value (both are valid)
- On selection from dropdown, set value and close dropdown
- Escape closes dropdown
- Click outside closes dropdown (useRef pattern from existing SearchBar)
- Keyboard: arrow keys to navigate options, Enter to select highlighted option

Styling: Follow existing patterns (bg-white, border-gray-200, rounded-lg, text-base, min-height 44px for touch targets, z-50 for dropdown, blue-600 focus ring). Error state with red border and error message below.
