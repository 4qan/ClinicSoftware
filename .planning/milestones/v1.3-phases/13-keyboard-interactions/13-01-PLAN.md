  <name>Task 1: Create useAutocompleteKeyboard hook and write tests for ComboBox keyboard fixes</name>
  <files>src/hooks/useAutocompleteKeyboard.ts, src/__tests__/ComboBox.test.tsx, src/components/ComboBox.tsx</files>
  <behavior>
    - Arrow Down opens dropdown when closed, sets highlight to 0
    - Arrow Down when open moves highlight forward, wraps to 0 at end
    - Arrow Up when open moves highlight backward, wraps to last at start
    - Enter with highlighted item selects it and closes dropdown
    - Enter with no highlight (highlightIndex=-1) selects first filtered item
    - Enter with no results does nothing
    - Escape closes dropdown without blurring input (focus stays on input)
    - Tab with highlighted item confirms selection, does NOT preventDefault (browser advances focus)
    - Tab with no highlight does nothing special (default browser behavior)
    - highlightIndex resets to -1 when items list changes
  </behavior>
  <action>
    1. Create `src/hooks/useAutocompleteKeyboard.ts`:
       - Generic hook: `useAutocompleteKeyboard<T>({ items: T[], isOpen: boolean, onSelect: (item: T) => void, onClose: () => void, onOpen?: () => void })`
       - Returns `{ highlightIndex, setHighlightIndex, handleKeyDown }`
       - Implements all keyboard behaviors: ArrowDown, ArrowUp, Enter (with enter-selects-first), Escape (no blur), Tab (no preventDefault)
       - Reset highlightIndex to -1 when items change (useEffect)

    2. Add keyboard behavior tests to `src/__tests__/ComboBox.test.tsx` (append to existing file):
       - Test each behavior listed above using @testing-library/user-event
       - Verify Escape does NOT blur (document.activeElement should still be the input)
       - Verify Tab does NOT call preventDefault (use spy on KeyboardEvent)

    3. Refactor ComboBox.tsx to use the hook:
       - Replace inline handleKeyDown with `useAutocompleteKeyboard({ items: filtered, isOpen, onSelect: handleSelect, onClose: close, onOpen: () => { setIsOpen(true); setHighlightIndex(0) } })`
       - Remove the `highlightIndex` useState (comes from hook now)
       - Remove `inputRef.current?.blur()` from Escape path (handled by hook not calling blur)
       - Add new optional prop `inputRef?: React.RefObject<HTMLInputElement | null>` so parent components can access the input. Use internal ref if not provided: `const internalRef = useRef<HTMLInputElement>(null); const ref = inputRef || internalRef`
       - Keep all existing visual rendering, click-outside, scroll-into-view logic unchanged

    Run: `npx vitest run src/__tests__/ComboBox.test.tsx`
  </action>
  <verify>
    <automated>npx vitest run src/__tests__/ComboBox.test.tsx</automated>
  </verify>
  <done>useAutocompleteKeyboard hook created and exported. ComboBox uses the hook. All keyboard behaviors pass tests. Escape does not blur. Tab does not preventDefault. ComboBox accepts optional inputRef prop. No visual changes to ComboBox rendering.</done>
