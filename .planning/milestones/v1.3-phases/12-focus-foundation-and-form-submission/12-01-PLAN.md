   - Class audit: read all .tsx files in src/components/ and src/pages/ and assert NONE contain `focus:ring-`, `focus:outline-none`, or `focus:border-` in className strings. Use `import.meta.glob` or fs to load file contents. This test will fail initially (RED).

   Note: The class audit test uses static file analysis (not runtime DOM), which works reliably in jsdom. Do NOT attempt to test :focus-visible pseudo-class behavior at runtime (jsdom doesn't support it).

2. Add the global focus-visible rule to `src/index.css` after the existing `@import "tailwindcss"` line and before the body rule:

```css
@layer base {
  :focus-visible {
    outline: 2px solid var(--color-blue-600);
    outline-offset: 2px;
  }

  :focus:not(:focus-visible) {
    outline: none;
  }
}
```

Use `var(--color-blue-600)` (TailwindCSS v4 design token). If it doesn't resolve at build time, try `theme(--color-blue-600)`. This keeps the color design-system-linked per user decision.

3. Strip ALL `focus:ring-*`, `focus:outline-none`, `focus:border-*` utility classes from className strings in every file listed below. Remove ONLY the focus-related classes; keep all other classes intact.
