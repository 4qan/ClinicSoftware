## Legacy-Install Inference (D-04)

`getDeploymentMode()` resolves the stored value with three branches:

1. Stored value is `'solo'` or `'networked'` → return it
2. No stored `deploymentMode` AND `couchUrl` exists → infer `'networked'`, **persist to localStorage**, return `'networked'`
3. Otherwise → return `'solo'` (default)

**Evidence (Test 2 — passing):**

```typescript
it('infers and persists "networked" when only couchUrl is set (legacy install)', () => {
  setCouchUrl('http://192.168.1.100:5984')
  expect(getDeploymentMode()).toBe('networked')
  expect(localStorage.getItem('deploymentMode')).toBe('networked')
})
```

This guarantees a Phase 22 install (which has `couchUrl` set but no `deploymentMode` key) is never silently downgraded to solo on first boot of the new code.

## Implementation Notes

- **No try/catch on the setter** (project pattern — assume localStorage is available; tests use jsdom). Getter is defensive on `JSON.parse` to survive corrupted storage.
- **Whitelist parser** on `deploymentMode`: any value other than the two canonical strings (e.g., `'xyz'`, injected garbage) collapses to the safe default `'solo'`. Verified by Test "falls back to 'solo' when stored value is invalid".
- **Module split** from PATTERNS guidance: localSettings.ts stayed at 40 lines, soloCredentials.ts is a clean 38-line sibling. Future PBKDF2 envelope changes ship without touching deploymentMode accessors.
- **Plan structure adaptation:** plan declared three TDD tasks (1: localSettings extend, 2: soloCredentials create, 3: tests). Because all three share one test file, executed as: RED commit (test file, both modules missing → import-resolve failure), then sequential GREEN commits for Task 1 and Task 2. Task 3 is satisfied by the RED commit (test file is committed, contains all 11 tests, all pass after Task 2). This preserves the TDD invariant (failing test before implementation) without duplicating commits.

## Threat Mitigations Applied

| Threat ID | Mitigation | Evidence |
|-----------|------------|----------|
| T-22.1-01 (Tampering deploymentMode) | Whitelist parser; non-canonical values fall back to `'solo'` | Test "falls back to 'solo' when stored value is invalid" |
| T-22.1-02 (Info disclosure soloCredentials) | Stored format never plaintext (Plan 02 enforces PBKDF2 hash) | Module shape forces `salt + hash + iterations`; no `password` field |
| T-22.1-03 (DoS via malformed JSON) | `JSON.parse` wrapped in try/catch; getter returns `null` | Test "returns null on malformed JSON (defensive)" |
| T-22.1-04 (Spoofing via legacy inference) | Inference only fires when `couchUrl` exists; persists deterministically | Test "infers and persists 'networked' when only couchUrl is set" |

`security_block_on: high` — no HIGH-severity unmitigated threats.
