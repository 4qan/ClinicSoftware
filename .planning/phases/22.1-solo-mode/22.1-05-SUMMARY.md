# Phase 22.1 Plan 05: Solo-mode UI Chrome Gating Summary

Hide all networked-only UI surfaces in solo mode (LoginPage URL form + "Change server address" button, Sidebar SyncIndicator + role label, Header role chip), neutralize LoginPage placeholders to "Username"/"Password", and add `data-testid="sync-indicator"` for deterministic test queries — all four components consume the same `isSolo = getDeploymentMode() === 'solo'` flag, networked path is byte-equivalent to Phase 22.

## W5 Rationale (data-testid)

Plan 07's workflow test asserts SyncIndicator is absent in solo. Without a testid, the assertion would rely on text matching (`Synced` / `Syncing` / `Disconnected` / `Offline`). That works, but a missing-text result is ambiguous: did Plan 04 omit `SyncProvider` (causing `useSyncContext` to throw before SyncIndicator renders)? Or did Plan 05 hide SyncIndicator? With `queryByTestId('sync-indicator')`:
- `null` -> correctly hidden by Plan 05 gate.
- non-null with no text -> regression in Plan 05 (gate missing).
- crash before render -> Plan 04 regression (provider tree).

The testid disambiguates these failure modes for the verifier.
