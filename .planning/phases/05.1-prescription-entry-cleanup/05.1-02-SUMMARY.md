# Phase 05.1 Plan 02: Drug Display Split + Amber Indicator Summary

Split drug display into search-result (full detail) vs. selected (brand name only), added amber border indicator on Frequency/Duration ComboBoxes for non-standard values.

## What Was Done

### Task 1: TDD RED - Tests and Utility (631d362)

Created `src/utils/drugFormatters.ts` with two functions extracted from the inline `formatDrugDisplay`:
- `formatDrugSearchResult(drug)`: returns "Brand (Salt Strength Form)"
- `formatDrugSelected(drug)`: returns brand name only

Created test files:
- `src/__tests__/ComboBox.test.tsx`: 6 tests covering amber border, non-blocking, no hint text
- `src/__tests__/MedicationEntry.test.tsx`: 5 tests for formatter functions

ComboBox tests failed (RED) because `showCustomIndicator` prop didn't exist yet.

### Task 2: GREEN - Implementation (42011ca)

**ComboBox.tsx:**
- Added `showCustomIndicator?: boolean` prop
- Computed `isCustomValue` via case-insensitive option match
- Applied `border-amber-400` class when custom value detected (error border takes precedence)

**MedicationEntry.tsx:**
- Replaced inline `formatDrugDisplay` with imported `formatDrugSearchResult`/`formatDrugSelected`
- `handleSelectDrug` now sets `drugQuery` to brand name only via `formatDrugSelected`
- Dropdown still shows full detail via `formatDrugSearchResult`
- Added `showCustomIndicator` to Frequency and Duration ComboBoxes

All tests passed (GREEN). Zero TypeScript errors.

### Task 3: Review Feedback Fixes (8c4afa9)

Applied user feedback:
- Removed "Custom value" hint text from ComboBox entirely (border-only, no layout shift)
- Removed `showCustomIndicator` from Qty ComboBox (quantity is inherently freeform)
- Updated tests to match

## Deviations from Plan

### Review-Driven Changes

**1. Qty field excluded from amber indicator**
- **Found during:** Task 3 (checkpoint review)
- **Issue:** Quantity is freeform; showing amber for "5" when only "1", "2" are predefined options is misleading
- **Fix:** Removed `showCustomIndicator` from Qty ComboBox

**2. "Custom value" hint text removed**
- **Found during:** Task 3 (checkpoint review)
- **Issue:** Text below input causes layout shift and breaks field alignment
- **Fix:** Removed hint text element entirely; amber border alone is sufficient signal
