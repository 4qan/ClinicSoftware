# Plan 01: Visit Flow UX and Settings Redesign

## Goal
Make the visit form always-visible with inline patient creation, redesign Settings into categories, add Clinic Info settings, and add Print link to VisitCard.

## must_haves
- [ ] Visit form sections (clinical notes, prescription, Rx notes) render on page load as disabled, enable when patient selected, no layout shift (VISIT-02)
- [ ] Search dropdown shows "Create '[query]' as new patient" when no results found and query >= 2 chars (VISIT-03)
- [ ] Clicking "Create" expands inline registration form, patient auto-selected on submit, no page navigation (VISIT-01)
- [ ] Settings page uses category tabs (Account, Medications, Clinic Info) instead of flat layout
- [ ] Clinic Info settings (doctor name, clinic name, address, phone, footer text) saved to Dexie settings table
- [ ] VisitCard shows a "Print" link navigating to `/visit/:id/print` (PRINT-04 entry point)

## Tasks

1. Open `src/pages/NewVisitPage.tsx`
2. Remove the `{selectedPatient && ...}` conditional wrappers around the Clinical Notes textarea, Prescription section (MedicationSection), Rx Notes textarea, and the Save/Action bar

   - Use a `fieldset` wrapper with `disabled` prop for clean disabling of form groups
4. The Visit History section can either show "Select a patient to see visit history" placeholder text or remain conditionally rendered (it has no meaningful empty state)
5. Verify no layout shift: the page height and structure should be identical before and after patient selection
</steps>

<task id="03-01-03" title="Inline patient creation in NewVisitPage">
<context>VISIT-01 and VISIT-03: when search finds no patients, show a "Create [query] as new patient" option. Clicking it expands the PatientRegistrationForm inline below the search, pre-filling the name. On submit, the new patient is auto-selected and the form collapses.</context>
<steps>
1. Open `src/pages/NewVisitPage.tsx`
2. In the search results dropdown, when results are empty and query >= 2 chars:
   - Replace the current "No patients found" + "Register New Patient" link with a button: "Create '[query]' as new patient"
   - Remove the link to `/register` from the no-results state
3. Add state: `showInlineRegistration: boolean` (default false)
4. When the "Create" button is clicked:
   - Set `showInlineRegistration = true`
   - Close the search dropdown
   - Parse the search query: first word = firstName, remaining = lastName
5. Render `PatientRegistrationForm` below the search input when `showInlineRegistration` is true:
   - Pass `initialFirstName` and `initialLastName` from parsed query
   - Pass `compact={true}` for tighter layout
   - Pass `submitLabel="Create & Select"`
   - `onSubmit`: call `registerPatient()`, then set the returned patient as `selectedPatient`, set `showInlineRegistration = false`, clear search query
   - `onCancel`: set `showInlineRegistration = false`
6. When inline form is visible, hide the search input (or disable it) to prevent conflicting interactions
7. Test the full flow: search for non-existent name, click create, fill remaining fields, submit, verify patient is auto-selected and visit form sections are now enabled
</steps>
