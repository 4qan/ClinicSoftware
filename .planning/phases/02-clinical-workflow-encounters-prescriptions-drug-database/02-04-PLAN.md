4. Add one-time dedup cleanup function:
```ts
export async function deduplicateExistingDrugs(): Promise<void> {
  const flag = await db.settings.get('drugsDeduped')
  if (flag) return
  const all = await db.drugs.toArray()
  const seen = new Map<string, string>()
  const toDelete: string[] = []
  for (const d of all) {
    const key = `${d.brandNameLower}|${d.saltNameLower}|${d.form}|${d.strength}`
    if (seen.has(key)) {
      toDelete.push(d.id)
    } else {
      seen.set(key, d.id)
    }
  }
  if (toDelete.length > 0) await db.drugs.bulkDelete(toDelete)
  await db.settings.put({ key: 'drugsDeduped', value: true })
}
```

**Changes in drugs.ts (searchDrugs dedup):**
Replace id-based dedup with composite key:
```ts

```

**Changes in App.tsx:**


### Task 4: Relocate "New Visit" to header CTA + remove from sidebar

**Files:** `src/components/Sidebar.tsx`, `src/components/AppLayout.tsx`

**Context:** "New Visit" as a sidebar nav item is wrong hierarchy (it's an action, not a destination). Research on clinical EMR UX (Jane App, SimplePractice, Cliniko) confirms: global creation actions belong in the header as persistent CTAs. With exactly 2 primary actions (New Patient, New Visit), dual header buttons is the optimal pattern: zero extra clicks, always visible, standard clinical software convention.

**Changes in Sidebar.tsx:**
- Remove the "New Visit" entry from navItems array. Sidebar should only contain navigation destinations: Home, Patients, Settings.

**Changes in AppLayout.tsx:**
- Add "New Visit" as a primary CTA button (filled blue, with icon) in the sticky header bar, next to the existing "Register Patient" button.
- **Visual hierarchy flip:** "New Visit" becomes the primary (filled) button since it's the higher-frequency action (doctors create 30+ visits/day). "Register Patient" gets demoted to outlined/secondary style.
- Both buttons link to their respective pages (`/visit/new` and registration flow).
- **Mobile (<640px):** Collapse both buttons to icon-only (drop text labels) to save horizontal space alongside the search bar.

**Verify:**
- Sidebar shows Home, Patients, Settings, Log Out only.
- Header shows two CTAs: "New Visit" (primary/filled) and "Register Patient" (secondary/outlined).
- Both buttons work and navigate correctly.
- On narrow viewport, buttons show icons only.
