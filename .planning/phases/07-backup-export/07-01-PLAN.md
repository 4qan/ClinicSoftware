**1. Update `package.json`:** Change `"version": "0.0.0"` to `"version": "1.1.0"`.

**2. Update `vite.config.ts`:** Add `define` block to inject app version at build time:
- Import `{ readFileSync }` from `fs` (or use a synchronous require/import of package.json)
- Read version from package.json
- Add `define: { '__APP_VERSION__': JSON.stringify(version) }` to the config
- Note: vitest.config.ts also needs the same `define` block so tests can access `__APP_VERSION__`

**3. Create `src/utils/backup.ts`:**
```typescript
declare const __APP_VERSION__: string;

export interface BackupMetadata {
  appName: string;
  exportDate: string;
  appVersion: string;
  schemaVersion: number;
  tables: Record<string, number>;
}

export interface BackupFile {
  metadata: BackupMetadata;
  data: Record<string, unknown[]>;
}
```
- `exportDatabase(): Promise<BackupFile>`: Iterate `db.tables`, call `.toArray()` on each, build `data` and `counts` objects. Return `{ metadata: { appName: 'ClinicSoftware', exportDate: new Date().toISOString(), appVersion: __APP_VERSION__, schemaVersion: db.verno, tables: counts }, data }`.
- `downloadBackup(backup: BackupFile): string`: `JSON.stringify(backup, null, 2)`, create Blob, `URL.createObjectURL`, create anchor, set `.download` to `ClinicSoftware-backup-${dateStr}.json` where dateStr is `new Date().toISOString().slice(0, 10)`. Click anchor, setTimeout cleanup (revokeObjectURL + removeChild). Return filename.

- Test "metadata tables has correct counts": seed 2 patients, assert `metadata.tables.patients === 2`.
- Test "download triggers file creation": mock `URL.createObjectURL`, `URL.revokeObjectURL`, `document.createElement`. Call `downloadBackup(backup)`. Assert filename matches `ClinicSoftware-backup-YYYY-MM-DD.json` pattern. Assert `URL.createObjectURL` was called with a Blob. Assert anchor `.click()` was called.
