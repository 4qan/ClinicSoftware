**1. Create `src/components/DataSettings.tsx`:**

State: `isExporting` (boolean), `lastBackup` (string | null, loaded on mount).

On mount: read `lastBackupDate` from `db.settings.get('lastBackupDate')`. If exists, set as `lastBackup`.

Export handler (`handleExport`):
1. Set `isExporting = true`
2. Try: call `exportDatabase()`, then `downloadBackup(backup)` (returns filename)
3. Store `lastBackupDate` in settings: `db.settings.put({ key: 'lastBackupDate', value: new Date().toISOString() })`
4. Update local `lastBackup` state
5. Show success toast: `showToast('success', \`Backup saved: ${filename} (${backup.metadata.tables.patients} patients, ${backup.metadata.tables.visits} visits)\`)`
6. Catch: `showToast('error', 'Backup failed. Please try again.')`
7. Finally: `isExporting = false`

UI layout (follows existing settings card pattern with `bg-white border border-gray-200 rounded-lg p-6`):
- `<h3>` "Backup & Restore" heading
- `<p>` brief description: "Export your clinic data to a backup file. You can restore from this file if you ever need to recover your data."
- "Export Backup" button (blue, same style as other settings CTAs). Disabled + "Exporting..." text when `isExporting`.
- Indeterminate progress bar below button (visible only when `isExporting`): simple animated Tailwind bar with `animate-pulse` or a CSS stripe animation on a `bg-blue-500` bar.
- `<p>` "Last backup: {formatted date}" or "Last backup: Never". Format date as locale string (e.g., "Mar 10, 2026, 3:45 PM").
