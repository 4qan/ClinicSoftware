# 05-02: Natural Language Urdu Instructions Column

## Problem

The current medication table uses 8 columns (#, Brand, Salt, Strength, Form, Dosage, Freq, Duration). On A5 paper this is cramped and hard to read, especially with bilingual headers on every column. Form, Dosage, Frequency, and Duration are closely related and read more naturally as a single Urdu sentence.

## Approach

Consolidate the four instruction-related columns into one "Instructions" column that renders a natural Urdu sentence. The sentence structure varies by medication form category since "take 1 tablet" and "apply thin layer" require different verb patterns. An English equivalent renders below in smaller text for bilingual readability.

New column layout: **# | Brand Name | Salt | Strength | Instructions** (5 columns).

## Form Categories

| Category | Forms | Sentence pattern (Urdu) |
|----------|-------|------------------------|
| Oral solid | Tablet, Capsule, Sachet, Powder | `{dosageUrdu} {frequencyUrdu} {durationUrdu} کے لیے` |
| Liquid | Syrup, Suspension, Solution | `{dosageUrdu} {frequencyUrdu} {durationUrdu} کے لیے` |
| Topical | Cream, Ointment, Gel, Patch | `{dosageUrdu} {frequencyUrdu} {durationUrdu} کے لیے` |
| Drops | Drops | `{dosageUrdu} {frequencyUrdu} {durationUrdu} کے لیے` |
| Injection | Injection | `{dosageUrdu} {frequencyUrdu} {durationUrdu} کے لیے` |
| Inhaler | Inhaler, Spray | `{dosageUrdu} {frequencyUrdu} {durationUrdu} کے لیے` |
| Suppository | Suppository | `{dosageUrdu} {frequencyUrdu} {durationUrdu} کے لیے` |

The dosage translations already contain form-aware verbs (e.g., "1 گولی" for tablets, "پتلی تہہ لگائیں" for topical, "1 قطرہ" for drops), so the sentence pattern itself is uniform: `{dosageUrdu} {frequencyUrdu} {durationUrdu} کے لیے`. The form-awareness comes from the dosage translation lookup, not the template. However, the category map is still needed for two reasons:

1. **Duration suffix logic**: "Ongoing" (جاری رکھیں) and "As needed" (ضرورت کے مطابق) should NOT get the "کے لیے" suffix since they are self-contained phrases.
2. **English equivalent formatting**: Different categories produce different English sentence patterns (e.g., "Take 1 tablet..." vs "Apply thin layer..." vs "Instill 1 drop...").

## Fallback Strategy

If any of dosage, frequency, or duration has no Urdu translation (toUrdu returns the original English string unchanged), fall back to rendering the four values as separate English entries in the Instructions cell rather than producing a broken half-Urdu sentence.

Detection: compare `toUrdu(value) === value` for each component. If any returns true (passthrough), use fallback.

<task id="05-02-01" title="Add buildUrduInstruction() to translations.ts">
<description>
Add to `src/constants/translations.ts`:

1. Define a `MedicationForInstruction` interface with the fields needed: `{ form: string; dosage: string; frequency: string; duration: string }`.

2. Define a `FORM_CATEGORY` map that classifies each of the 16 forms into categories: `oral`, `liquid`, `topical`, `drops`, `injection`, `inhaler`, `suppository`. Default to `oral` for unknown forms.

3. Define an `ENGLISH_VERB_PREFIX` map per category for the English equivalent line:
   - oral: "Take"
   - liquid: "Take"
   - topical: "Apply"
   - drops: "Instill"
   - injection: "Administer"
   - inhaler: "Inhale"
   - suppository: "Insert"

4. Export `buildUrduInstruction(med: MedicationForInstruction): { urdu: string; english: string } | null`.

Logic:
- Look up `toUrdu(med.dosage)`, `toUrdu(med.frequency)`, `toUrdu(med.duration)`.
- If ANY of these three is a passthrough (equals the English input), return `null` (signals fallback).
- Determine if duration is a self-contained phrase: check if `med.duration` is `'Ongoing'` or `'As needed'`. If so, omit "کے لیے" suffix.
- Urdu sentence: `${dosageUrdu} ${frequencyUrdu} ${durationUrdu}` (+ ` کے لیے` unless self-contained duration).
- English sentence: `${verbPrefix} ${med.dosage}, ${med.frequency}, ${med.duration}`.
- Return `{ urdu, english }`.

5. Add `'Instructions'` key to `columnHeadersUrdu` with value `'ہدایات'`.
6. Remove the now-unused individual column header entries for `'Form'`, `'Dosage'`, `'Freq'`, `'Duration'` from `columnHeadersUrdu`.
</description>
